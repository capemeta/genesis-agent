# 同轮工具并行调度设计

本文是 Genesis Agent **同轮 sibling tool call** 并行执行与回传模型的权威说明。文件系统资源锁细节仍以 [`文件系统设计方案.md`](文件系统设计方案.md) §8 为准；子智能体槽位限流以 [`子智能体设计.md`](子智能体设计.md) §7.1 为准。

## 1. 目标与第一性原理

| 问题 | 约束 |
| --- | --- |
| LLM 一轮可返回多个 tool_calls | 必须能安全并行独立只读/并发安全调用 |
| 写操作与状态变更 | 必须有 ordering barrier，避免乱序写与脏读 |
| 回传模型 | **整批 tool_result 齐了再进下一轮 LLM**（协议要求 tool_call 与 result 成对） |
| 失败传播 | sibling 失败后不应继续在错误上下文中写文件；未开始/未完成者回 sibling_error |
| 子智能体 fan-out | 复用同一调度器，不另造编排引擎；并发硬限在 SlotLimiter |

## 2. 与参考实现对齐

| 能力 | Kode-CLI | Codex | Genesis |
| --- | --- | --- | --- |
| 并行谓词 | `isConcurrencySafe(input)` | `supports_parallel_tool_calls` + RwLock | 静态 `ConcurrencySafe` + 可选 `ConcurrencyAssessor(params)` |
| Barrier | 非 safe 串行 | 非 parallel 持 write lock | `ConcurrencySafe=false` 串行 |
| 回传 LLM | 整批后 recursive query | `drain_in_flight` 后 follow-up | `Queue.Run` 结束后再拼 Messages |
| Sibling 失败 | `Sibling tool call errored` | 不强制取消 sibling | `ErrSiblingCanceled` + 同文案 |
| Task 并行 | Task 标 concurrency-safe | multi-agent 另套 | `Task.ConcurrencySafe=true`，槽位 `SlotLimiter` |
| MCP 并行 | 默认 false | hint / server opt-in | 默认 false；`read_only_hint` 或 assessor |

## 3. 运行时数据流

```text
LLM assistant message (N tool_calls)
  → Skill collision 改写 / Skill 独占轮特判
  → ResolveExecutionTraits(tool, params)   // 静态 + ConcurrencyAssessor
  → scheduler.Queue.Run
       · ConcurrencySafe 连续段：有界并行（默认 max=4）
       · 非 safe：barrier 串行
       · 任一主失败：取消未完成并发 sibling；后续未开始 → ErrSiblingCanceled
  → 按原顺序组装 ToolResult（失败仍带 stdout/摘要；sibling 用固定文案）
  → 全部写入 rc.Messages
  → 下一轮 LLM
```

**不变量**：UI/progress 可流式；**下一轮模型请求不得在 sibling 未齐时发出**。

## 4. 核心契约

### 4.1 并行谓词

```go
// 调度并行只看 ConcurrencySafe
isSiblingConcurrent = traits.ConcurrencySafe
```

- `ReadOnly` **不否决并行**，只影响 Gateway / 资源锁模式（读锁 vs 写锁）。
- 历史错误：`ReadOnly && ConcurrencySafe` 会把 `Task`（`ReadOnly=false, ConcurrencySafe=true`）误串行。

### 4.2 入参级评估（可选）

```go
type ConcurrencyAssessor interface {
    AssessConcurrency(ctx context.Context, params string) ConcurrencyAssessment
}
```

| 规则 | 说明 |
| --- | --- |
| 未实现 | 仅用 `GetInfo` / `DefaultTraits` |
| 解析失败 | 必须返回零值（`ConcurrencySafe=false`） |
| 可升级 | 如 `run_command` 只读命令、`search_mcp_tools` 无 promote |
| 可降级 | 静态 safe 但本轮入参不安全时覆盖为 false |

解析入口：`tool.ResolveExecutionTraits`（`react_loop.executeToolCalls` 调用）。

### 4.3 Sibling 失败

| 错误 | 含义 | 对 Run 的影响 |
| --- | --- | --- |
| 工具主失败 | 该 call 的真实错误 | 取消未完成 sibling；结果仍回模型 |
| `ErrSiblingCanceled` | 因 sibling 失败未执行/中断 | **不**中止 Run；内容为 Kode 同款文案 |
| `ErrRunAborted` / 父 ctx 取消 | 用户中止 | 中止整批与 Run |

文案常量：`scheduler.SiblingErrorContent` = `<tool_use_error>Sibling tool call errored</tool_use_error>`。

### 4.4 Skill 独占轮

`Skill` 网关与其它工具同轮时：**只执行 Skill**，其余回「跳过」说明。注入会改变可用工具集，禁止与业务工具并行。

## 5. 工具 traits 策略（现状）

| 工具 / 类别 | ConcurrencySafe | 备注 |
| --- | --- | --- |
| `read_file` / `grep` / `glob` / `list_dir` / `walk_dir` | true（只读） | 默认可并行 |
| `write_file` / `edit_file` / `apply_patch` | false | barrier |
| `Task` | true | 并行 fan-out；`max_concurrent` 硬限 |
| `Skill` / `run_skill_command` | false | 状态/环境变更 |
| `run_command` | 静态 false；assessor 可升级 | 仅分类为只读且非危险时并行 |
| `mcp__*` | 默认 false；`read_only_hint` → true | 对齐 Codex |
| `search_mcp_tools` | 静态 false；无 promote 时 assessor → true | promote 改 registry，不可并行 |
| skill 资源 list/read/search | true | 只读包内资源 |

## 7. 验证要点

- 同轮多个 `ConcurrencySafe` 工具真正重叠执行（见 `queue_test.go`）。
- `ReadOnly=false` 且 `ConcurrencySafe=true`（Task 形态）可并行。
- barrier 后工具不得抢跑。
- 主失败后后续 sibling 为 `ErrSiblingCanceled`，Run 不因此 abort。
- `run_command` / `search_mcp_tools` assessor 升级与降级单测。
- 回传顺序与 tool_call 顺序一致。

## 8. 明确未做 / 残余风险

| 项 | 状态 |
| --- | --- |
| Queue 内按 file/workspace `ResourceLock` 细粒度排队 | 工具内部 / Gateway 已有部分锁；**尚未**并入 Queue 调度主路径 |
| `run_command` 在 ShellAuto 未解析到具体 shell 时 | 保守不并行 |
| MCP 非只读的 server 级 opt-in 并行 | 未做（Codex 有）；默认串行更安全 |
| sibling 失败策略是否对「探索类软失败」过严 | 接受 Kode 语义；若产品要「并行 Task 互不取消」需另开配置 |
