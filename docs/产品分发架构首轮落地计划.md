# 产品分发架构首轮落地计划

## Summary

本计划是第一轮编码唯一参照。目标是落地产品独立打包边界，而不是实现 SQLite/PostgreSQL、文件系统 backend、sandbox、多 Agent 业务细节。

第一轮完成后应满足：

- `cmd/genesis-cli` 和 `cmd/genesis-enterprise` 可独立构建。
- CLI / Enterprise 分别通过自己的 `distributions/<product>/bootstrap` 装配。
- `internal/bootstrap` 只保留产品无关 shared builder。
- `internal/interfaces/cli` 不再直接依赖 `internal/bootstrap`。
- `scripts/check_product_isolation.ps1` 可检查 import graph。
- 旧兼容入口 `cmd/agent`、`cmd/api` 已移除，正式入口是唯一启动路径。

## Key Changes

### 1. Shared Builder

`internal/bootstrap` 提供产品无关构建 API：

```go
type BuildOptions struct {
    ConfigDir        string
    Quiet            bool
    RouteName        string
    DefaultAgentID   string
    DefaultAgentName string
    Profile          profilemodel.Profile
}

type RuntimeBundle struct {
    Config       *config.Config
    Logger       logger.Logger
    Tracer       trace.Tracer
    ToolRegistry tool.Registry
    MemoryStore  memory.ShortTermStore
    DefaultAgent *domain.Agent
    AgentService app.AgentService
}

func BuildAgentService(ctx context.Context, opts BuildOptions) (*RuntimeBundle, error)
```

默认规则：

- `RouteName` 为空时使用 `chat`。
- `DefaultAgentID` 为空时使用 `default-agent`。
- `DefaultAgentName` 为空时使用 `Genesis Agent`。
- 第一轮 memory store 仍使用 `inmemory.NewInMemoryStore()`。
- 不引入 SQLite/PostgreSQL/sandbox/OIDC/Wails。
- `internal/bootstrap` 不 import `distributions/*`。

### 2. Product Bootstrap

新增产品装配层：

- `distributions/cli/bootstrap`
- `distributions/enterprise/bootstrap`

两者暴露一致接口：

```go
func NewContainer(configDirRef *string, quiet bool) *Container
func (c *Container) Init(ctx context.Context) error
func (c *Container) Service() app.AgentService
func NewService(ctx context.Context, configDirRef *string, quiet bool) (app.AgentService, error)
```

CLI 第一轮使用默认 Agent：`default-agent` / `Genesis Agent`。
Enterprise 第一轮复用 shared builder，但使用：`enterprise-default-agent` / `Genesis Enterprise Agent`。

同时新增轻量骨架包：

- `distributions/local/doc.go`
- `distributions/desktop/doc.go`
- `distributions/enterprise/doc.go`

### 3. Product Entrypoints

新增：

- `cmd/genesis-cli/main.go`
- `cmd/genesis-enterprise/main.go`

旧兼容入口不再保留：`cmd/agent`、`cmd/api` 已移除。产品入口必须通过正式 `cmd/genesis-cli` 和 `cmd/genesis-enterprise` 进入。

### 4. CLI Factory Injection

`internal/interfaces/cli` 新增：

```go
type ServiceFactory func(ctx context.Context, configDirRef *string, quiet bool) (app.AgentService, error)
func ExecuteWithFactory(factory ServiceFactory) error
```

不再保留 `Execute()` 兼容入口。需要服务的命令必须通过 `ExecuteWithFactory` 注入产品 bootstrap，避免绕过产品装配根。

### 5. Product Isolation Script

新增 `scripts/check_product_isolation.ps1`，通过 `go list -deps` 检查：

CLI 禁止依赖：

- `genesis-agent/distributions/enterprise`
- `github.com/jackc/pgx`
- `github.com/lib/pq`
- `github.com/wailsapp/wails`

Enterprise 禁止依赖：

- `genesis-agent/distributions/local`
- `genesis-agent/distributions/desktop`
- `github.com/wailsapp/wails`

第一轮不禁止 CLI SQLite，因为未来 CLI SQLite 会通过单独入口或 build tag 控制；当前实现也不应引入 SQLite。

## Test Plan

```powershell
go test ./internal/...
go test ./cmd/genesis-cli/... ./distributions/cli/...
go test ./cmd/genesis-enterprise/... ./distributions/enterprise/...

go build -o bin/genesis-cli.exe ./cmd/genesis-cli
go build -o bin/genesis-enterprise.exe ./cmd/genesis-enterprise

powershell -ExecutionPolicy Bypass -File scripts/check_product_isolation.ps1
```

行为检查：

```powershell
bin/genesis-cli.exe tools
bin/genesis-cli.exe config
bin/genesis-enterprise.exe --help
```

## Acceptance Criteria

- 所有构建成功。
- CLI commands still initialize current tools/config behavior。
- Enterprise help and server initialization path remain equivalent to原 HTTP server 行为。
- Isolation script passes。
- `internal/interfaces/cli` has no import of `internal/bootstrap`。
- `internal/interfaces/cli` 不保留 `Execute()` 兼容入口。
- Product entries do not import the wrong distribution package。
- No SQLite/PostgreSQL/Wails/sandbox/OIDC dependency is introduced in this first round。

## Assumptions

- 第一轮只落地架构边界，不扩展业务能力。
- 不实现文件系统 backend。
- 不实现 SQLite/PostgreSQL。
- 不实现 sandbox client。
- 不实现 Wails desktop。
- 不实现多 Agent runtime。
- 保留当前 in-memory memory store 和现有工具注册逻辑。
- 不保留旧兼容入口，避免绕过产品 bootstrap。
