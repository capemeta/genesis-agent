# Genesis Agent 产品目录彻底迁移计划

> 目标：从 `distributions/* + internal/interfaces/* + 根 web` 升级为 `internal 共享内核 + products 独立产品 + shared 共享适配 + cmd 薄入口`。  
> 本文是目录迁移与边界收敛的执行计划，不拆仓库、不拆 Go module、不做业务能力扩展。

---

## 一、核心结论

当前 `distributions/` 方案已经能做到 Go import graph 层面的独立打包，但它仍有三个长期问题：

1. **产品代码散落**：CLI 命令/TUI 在 `internal/interfaces/cli`，Enterprise HTTP 在 `internal/interfaces/http`，Web 在根 `web/`，产品边界不完整。
2. **`internal` 语义膨胀**：`internal` 既像共享内核，又包含产品接入面，后续会把 CLI/Desktop/Enterprise 逻辑重新长回共享层。
3. **`distributions` 语义偏窄**：它更像“分发差异层”，但现在承载的是完整产品代码，名称会误导后续开发。

最终目标结构：

```text
cmd/
  genesis-cli/
  genesis-desktop/
  genesis-enterprise/

internal/                    # 共享 Agent 内核，不属于任何产品
  app/
  bootstrap/                 # shared builder，不含产品语义
  capabilities/
  domain/
  platform/
  runtime/

products/                    # 独立产品
  cli/
    bootstrap/
    internal/
      approval/
      command/
      profile/
      tui/
      persistence/
        file/
        sqlite/

  desktop/
    bootstrap/
    internal/
      approval/
      interfaces/
        http/
        wails/
      multiagent/
      native/
      persistence/
        sqlite/
      profile/
    web/

  enterprise/
    bootstrap/
    internal/
      admin/
      audit/
      auth/
      iam/
      interfaces/
        http/
        sse/
      persistence/
        postgres/
          migrations/
          query/
          dbgen/
      profile/
      rbac/
      sandbox/
      tenant/
    web/

shared/                      # 非内核，但可被多个产品复用的适配实现
  local/
    execution/
    filesystem/
    pathresolver/
    sandbox/
```

设计原则：

- `internal/` 只放产品无关的 Agent 内核、能力契约、通用服务和平台基础设施。
- `products/<product>/bootstrap` 是该产品唯一对外暴露的 Composition Root。
- `products/<product>/internal` 放产品私有实现，通过 Go `internal` 规则辅助隔离。
- `shared/local` 放 CLI/Desktop 共享的本地主机适配，不属于共享内核，也不属于 Enterprise。
- `cmd/<product>` 是薄入口，只 import 自己产品的 bootstrap。

## 二、最终依赖规则

### 2.1 允许方向

```text
cmd/<product>
  -> products/<product>/bootstrap

products/<product>/bootstrap
  -> internal/*
  -> products/<product>/internal/*
  -> shared/local/*                 # 仅 CLI/Desktop 默认允许

products/<product>/internal/*
  -> internal/*
  -> shared/local/*                 # 仅 CLI/Desktop 默认允许

shared/local/*
  -> internal/capabilities/* contract/model
  -> internal/platform/* 可复用基础设施
```

### 2.2 禁止方向

```text
internal/*                          -X-> products/*
internal/*                          -X-> shared/local/*
shared/local/*                      -X-> products/*
products/cli/*                      -X-> products/desktop/*
products/cli/*                      -X-> products/enterprise/*
products/desktop/*                  -X-> products/cli/*
products/desktop/*                  -X-> products/enterprise/*
products/enterprise/*               -X-> products/cli/*
products/enterprise/*               -X-> products/desktop/*
products/enterprise/*               -X-> shared/local/*
cmd/<product>                       -X-> internal/*  # 除非极少数标准版本变量，默认禁止
```

### 2.3 产品入口直接 import 白名单

`cmd/genesis-cli`：

```text
genesis-agent/products/cli/bootstrap
```

`cmd/genesis-desktop`：

```text
genesis-agent/products/desktop/bootstrap
```

`cmd/genesis-enterprise`：

```text
genesis-agent/products/enterprise/bootstrap
```

产品入口不再直接 import `internal/interfaces/cli` 或 `internal/interfaces/http`。CLI command 和 Enterprise HTTP 都应迁入对应产品目录。

## 三、迁移映射

### 3.1 后端 Go 目录

| 当前目录 | 目标目录 | 说明 |
|---|---|---|
| `distributions/cli/bootstrap` | `products/cli/bootstrap` | CLI composition root，对外暴露给 `cmd/genesis-cli` |
| `distributions/cli/profile` | `products/cli/internal/profile` | CLI profile 是产品私有实现 |
| `distributions/cli/approval` | `products/cli/internal/approval` | CLI 终端审批 requester 是产品私有实现 |
| `internal/interfaces/cli` | `products/cli/internal/command` | CLI 命令、root、run、tools、config、version |
| `internal/interfaces/cli/tui` | `products/cli/internal/tui` | CLI TUI 是 CLI 产品私有 UI |
| `distributions/enterprise/bootstrap` | `products/enterprise/bootstrap` | Enterprise composition root |
| `distributions/enterprise/profile` | `products/enterprise/internal/profile` | Enterprise profile 是产品私有实现 |
| `internal/interfaces/http` | `products/enterprise/internal/interfaces/http` | 当前 HTTP server/router/handler 属于 Enterprise API |
| `distributions/local/fs_backend` | `shared/local/filesystem` | CLI/Desktop 共享本地 FS adapter |
| `distributions/local/pathresolver` | `shared/local/pathresolver` | CLI/Desktop 共享本地 path resolver |
| `distributions/local/doc.go` | `shared/local/doc.go` | 本地主机共享适配说明 |
| `distributions/desktop` | `products/desktop` | 桌面产品骨架 |
| `internal/interfaces/desktop` | `products/desktop/internal/interfaces` | 桌面接入面不属于共享内核 |

### 3.2 前端与部署资产

| 当前目录 | 目标目录 | 说明 |
|---|---|---|
| `web` | `products/enterprise/web` | 当前 Web 控制台默认归 Enterprise |
| `api` | `products/enterprise/api` 或保留根 `api` | 若是 OpenAPI/公共 API 契约可保留根；若是企业 HTTP 专属文档迁入 Enterprise |
| `deploy/sql` | `products/enterprise/internal/persistence/postgres/migrations` | 企业 PostgreSQL 迁移归 Enterprise |
| `deploy/docker` | `deploy/docker` | 若是全仓通用部署模板可保留 |
| `deploy/k8s` | `deploy/k8s` | 若只服务 Enterprise，后续可迁 `products/enterprise/deploy/k8s` |

前端迁移要同步修改：

- `products/enterprise/web/package.json` 中脚本路径。
- CI 中 web 工作目录。
- 文档中所有 `web/` 引用。
- 若 Desktop 前端未来独立，不从 Enterprise Web 直接复用；共享组件等复用事实出现后再抽 `products/shared/web` 或 `web-shared`。

### 3.3 不迁移的目录

| 目录 | 原因 |
|---|---|
| `internal/capabilities/*` | 共享能力域，保持产品无关 |
| `internal/runtime/*` | Agent Runtime 内核 |
| `internal/domain/*` | 核心领域模型 |
| `internal/platform/*` | 产品无关基础设施 |
| `internal/app/*` | 共享应用编排 |
| `internal/bootstrap/*` | shared builder，不含产品语义 |
| `docs` | 仓库级文档 |
| `scripts` | 仓库级脚本 |
| `test` | 仓库级集成测试 |

## 四、目标包职责

### 4.1 `internal`

`internal` 是共享 Agent 内核，只能表达产品无关能力：

- Agent runtime、strategy、action、stop、prompt。
- capability contract/model/service。
- 产品无关 adapter，例如 LLM adapter、通用 HTTP client、genesis-sandbox API client 的纯 SDK 包。
- domain model。
- platform 基础设施。
- shared builder。

`internal` 不允许出现：

- CLI cobra command。
- Bubble Tea TUI。
- Wails/native desktop adapter。
- Enterprise HTTP handler。
- RBAC/OIDC/Postgres 具体企业适配。
- 本地主机 FS / `exec.Command` / OS sandbox 具体实现。

### 4.2 `products/cli`

CLI 是独立产品，包含：

- `bootstrap`：选择 CLI profile、本地 FS、本地 path resolver、CLI approval requester、CLI persistence。
- `internal/command`：cobra root、chat、run、tools、config、version。
- `internal/tui`：终端 UI。
- `internal/approval`：终端同步 approval requester。
- `internal/profile`：CLI 默认能力集。
- `internal/persistence`：文件或可选 SQLite。

CLI 可依赖 `shared/local`，但不能依赖 Enterprise、Desktop。

### 4.3 `products/desktop`

Desktop 是独立产品，包含：

- `bootstrap`：桌面产品装配。
- `internal/native`：OS 原生能力。
- `internal/interfaces/wails`：Wails binding。
- `internal/interfaces/http`：桌面本地 API，如果存在。
- `internal/approval`：GUI 弹窗、native picker、会话授权。
- `internal/multiagent`：桌面投影、画布/任务板适配。
- `web`：桌面前端。

Desktop 可依赖 `shared/local`，但不能依赖 CLI、Enterprise。

### 4.4 `products/enterprise`

Enterprise 是独立产品，包含：

- `bootstrap`：企业产品装配。
- `internal/interfaces/http`：HTTP server/router/handler。
- `internal/interfaces/sse`：SSE 接入。
- `internal/auth` / `iam` / `rbac` / `tenant`：企业治理能力具体实现。
- `internal/audit`：企业审计适配。
- `internal/sandbox`：企业 sandbox 上下文、endpoint/credential 注入、治理策略。
- `internal/persistence/postgres`：PostgreSQL 迁移、查询、dbgen。
- `web`：企业控制台。

Enterprise 默认不能依赖 `shared/local`，避免把本地主机 FS/执行能力打进企业二进制。

### 4.5 `shared/local`

`shared/local` 是 CLI/Desktop 共享的本地主机适配层：

- 本地 FileSystemBackend。
- 本地 PathResolver。
- 本地 CommandRunner。
- 本机 OS sandbox wrapper。

它可以依赖 `internal/capabilities/*/contract` 和 `internal/capabilities/*/model`，但不能依赖任何产品目录。

## 五、迁移执行清单

> 虽然执行可以拆成多个提交，但目标不是“阶段性折中”。每个提交都必须朝最终结构收敛，不新增旧路径代码。

### 5.1 建立新目录骨架

创建：

```text
products/cli/bootstrap
products/cli/internal/{approval,command,tui,profile,persistence}
products/desktop/bootstrap
products/desktop/internal/{approval,interfaces,multiagent,native,persistence,profile}
products/enterprise/bootstrap
products/enterprise/internal/{admin,audit,auth,iam,interfaces,persistence,profile,rbac,sandbox,tenant}
shared/local/{filesystem,pathresolver,execution,sandbox}
```

### 5.2 迁移 CLI 产品

移动：

```text
distributions/cli/bootstrap        -> products/cli/bootstrap
distributions/cli/profile          -> products/cli/internal/profile
distributions/cli/approval         -> products/cli/internal/approval
internal/interfaces/cli/*.go       -> products/cli/internal/command
internal/interfaces/cli/tui        -> products/cli/internal/tui
```

修改 import：

```text
genesis-agent/distributions/cli/bootstrap
  -> genesis-agent/products/cli/bootstrap

genesis-agent/distributions/cli/profile
  -> genesis-agent/products/cli/internal/profile

genesis-agent/distributions/cli/approval
  -> genesis-agent/products/cli/internal/approval

genesis-agent/internal/interfaces/cli
  -> genesis-agent/products/cli/internal/command

genesis-agent/internal/interfaces/cli/tui/...
  -> genesis-agent/products/cli/internal/tui/...
```

修改 `cmd/genesis-cli/main.go`：

```go
import "genesis-agent/products/cli/bootstrap"

func main() {
    if err := bootstrap.Execute(context.Background()); err != nil {
        // 入口只处理退出码和错误打印
    }
}
```

推荐让 `products/cli/bootstrap` 对外暴露 `Execute(ctx)`，这样 `cmd/genesis-cli` 不需要 import command 包。

### 5.3 迁移 shared local

移动：

```text
distributions/local/fs_backend     -> shared/local/filesystem
distributions/local/pathresolver   -> shared/local/pathresolver
```

修改 CLI bootstrap import：

```text
genesis-agent/distributions/local/fs_backend
  -> genesis-agent/shared/local/filesystem

genesis-agent/distributions/local/pathresolver
  -> genesis-agent/shared/local/pathresolver
```

保留原则：

- `shared/local` 内可以 import `os/path/filepath/exec` 等本地主机 API。
- `internal/capabilities/filesystem/tool` 仍不能 import `shared/local` 或 `os/path/filepath`。

### 5.4 迁移 Enterprise 产品

移动：

```text
distributions/enterprise/bootstrap -> products/enterprise/bootstrap
distributions/enterprise/profile   -> products/enterprise/internal/profile
internal/interfaces/http           -> products/enterprise/internal/interfaces/http
web                                -> products/enterprise/web
```

修改 import：

```text
genesis-agent/distributions/enterprise/bootstrap
  -> genesis-agent/products/enterprise/bootstrap

genesis-agent/distributions/enterprise/profile
  -> genesis-agent/products/enterprise/internal/profile

genesis-agent/internal/interfaces/http
  -> genesis-agent/products/enterprise/internal/interfaces/http
```

推荐让 `products/enterprise/bootstrap` 对外暴露：

```go
func Execute(ctx context.Context) error
func NewContainer(configDirRef *string, quiet bool) *Container
func NewService(ctx context.Context, configDirRef *string, quiet bool) (app.AgentService, error)
```

`cmd/genesis-enterprise` 只调用 `bootstrap.Execute(ctx)`。

### 5.5 迁移 Desktop 骨架

移动或新建：

```text
distributions/desktop              -> products/desktop
internal/interfaces/desktop        -> products/desktop/internal/interfaces
cmd/genesis-desktop                -> import products/desktop/bootstrap
```

Desktop 当前如果只是骨架，也要有 `doc.go` 明确：

- 不 import CLI。
- 不 import Enterprise。
- 可 import `shared/local`。
- Wails adapter 只能在 `products/desktop/internal/interfaces/wails`。

### 5.6 删除旧目录

迁移完成后删除：

```text
distributions/
internal/interfaces/
web/                               # 已迁入 products/enterprise/web
```

删除前必须确认：

```powershell
rg "distributions/" .
rg "internal/interfaces" .
rg "genesis-agent/web|../web|web/" docs scripts products cmd internal
```

剩余命中只能出现在历史文档或迁移说明中；正式架构文档和代码不得再引用旧目录。

## 九、风险与处理

### 9.1 Go `internal` 可见性

`products/cli/bootstrap` 不能 import `products/cli/internal/...` 吗？

可以。Go 的 internal 规则允许父目录及其子目录访问：

```text
products/cli/bootstrap
products/cli/internal/command
```

两者都在 `products/cli` 树内，所以允许。

`cmd/genesis-cli` 不能 import `products/cli/internal/command`，这是预期；入口只能 import bootstrap。

### 9.2 Enterprise HTTP 迁移

当前 `cmd/genesis-enterprise` 直接 import `internal/interfaces/http`。迁移后应改成只 import `products/enterprise/bootstrap`，由 enterprise bootstrap 内部启动 HTTP server。

这会让 `cmd` 更薄，也避免 Enterprise 接口留在共享内核。

### 9.3 Web 迁移

根 `web/` 迁入 `products/enterprise/web` 会影响前端脚本、CI、文档和可能的 IDE 配置。迁移时不要改业务代码，只做路径移动和脚本工作目录修正。

### 9.4 测试缓存和模块缓存

当前仓库中出现 `.gomodcache`、`.testdata` 等本地开发目录。迁移目录时不要把这些目录纳入架构判断；必要时在 `.gitignore` 中明确忽略。

### 9.5 历史文档冲突

旧文档会大量出现 `distributions`。迁移完成后，正式架构文档必须统一口径；历史分析文档可以保留，但要在顶部标记“历史参考，不作为当前目标结构”。

## 十、完成定义

迁移完成必须同时满足：

- 代码中不再有 `genesis-agent/distributions/...` import。
- `distributions/` 目录不存在。
- `internal/interfaces/` 目录不存在，产品 interface 已迁入对应 `products/<product>/internal/interfaces` 或 CLI command/tui。
- 根 `web/` 已迁入 `products/enterprise/web`，或文档明确说明暂保留原因和迁移阻塞点。
- `cmd/<product>` 只 import 自己的 `products/<product>/bootstrap`。
- `internal/...` 不 import `products/...` 或 `shared/local/...`。
- `products/enterprise/...` 不 import `shared/local/...`。
- `scripts/check_product_isolation.ps1` 覆盖新边界。
- 所有验证命令通过。

## 十一、迁移后的开发规则

新增代码默认按以下问题决定位置：

1. 是否所有产品都需要，且不依赖具体 UI/DB/OS/沙箱实现？  
   是：放 `internal`。

2. 是否 CLI/Desktop 共用的本地主机 adapter，且 Enterprise 默认不能引入？  
   是：放 `shared/local`。

3. 是否只属于某个产品的 UI、bootstrap、profile、persistence、auth、approval、接口适配？  
   是：放 `products/<product>/internal` 或 `products/<product>/bootstrap`。

4. 是否是产品入口？  
   是：放 `cmd/<product>`，并保持薄壳。

5. 是否需要跨产品复用但又不是内核？  
   先不要抽象。只有出现两个产品真实复用且不污染 Enterprise/CLI 依赖时，再考虑放 `shared/<domain>`。
